처음 사용하거나 기준이 헷갈릴 때 확인하는 안내입니다.

이 보드의 목적

이 보드는 기존 관리 툴을 대체하지 않습니다. 기존 관리 툴은 업무 진행과 코멘트가 쌓이는 기준 자료로 사용합니다.

담당 기획자가 개발 기간 중 티켓 상태와 이슈를 짧게 남기고, 리더가 빌드 진행 상황을 빠르게 확인하기 위한 보조 도구입니다.

퇴근 전 업무 공유 문장을 새로 고민하지 않도록, 입력된 내용을 바탕으로 공유용 문장을 생성합니다.

티켓 등록부터 완료까지

1

티켓 리뷰

이번 빌드에 포함할 업무 항목을 확정합니다.

2

티켓 등록

업무 링크, 티켓명, 담당 기획자, 작업 담당/파트를 등록합니다.

3

개발 기간 확인

담당 기획자가 본인 티켓 진행 상황을 확인합니다.

4

상태 갱신 / 변동 없음

이슈가 있으면 상태를 갱신하고, 같으면 마지막 확인일만 갱신합니다.

5

완료 확인 대기

개발 완료로 보이면 QA/빌드 확인 대상으로 표시합니다.

6

완료

기획 확인 또는 빌드 확인이 끝나면 완료로 변경합니다.

매일 하는 업무 공유 루틴

1

오늘 확인

내 담당 티켓에서 이슈, 완료 확인 대기, 미확인 티켓을 우선 확인합니다.

2

변동 없음 / 상태 변경

상태가 같으면 확인 시간만 갱신하고, 이슈가 있으면 상태와 메모를 남깁니다.

3

일일 업무 공유

퇴근 전 내 담당 티켓 현황을 공유 문장으로 복사합니다.

4

메신저 공유

복사한 내용을 메신저에 붙여넣어 공유합니다.

체크 상태

정상

개발, 아트, 기획 흐름이 문제 없이 진행 중인 상태입니다.

이슈 있음

일정, 구현, 보상, 밸런스, 리소스 등에서 확인이 필요한 내용이 있는 상태입니다.

완료 확인 대기

개발이 완료되었거나 완료된 것으로 보여 기획 확인 또는 빌드 확인이 필요한 상태입니다.

완료

이번 단계에서 담당 기획자가 추가로 챙길 내용이 없는 상태입니다.

다음 액션

없음

개발, 아트, 기획 흐름이 정상적으로 진행 중이며 추가 확인이나 요청 사항이 없는 상태입니다.

개발 확인 필요

개발 완료 후 확인 중 이슈가 생겼거나, 구현 방식, 일정, 동작 차이 등 개발 쪽 확인이 필요한 상태입니다.

리더 확인 요청

범위, 우선순위, 일정, 제외 여부 등 리더 판단이 필요한 상태입니다.

QA/빌드 확인 필요

QA 결과나 실제 빌드에서 기획 의도대로 동작하는지 확인이 필요한 상태입니다.

아트 폴리싱 진행중

리소스, 연출, UI 비주얼, 이펙트 등 최종 느낌을 맞추는 중인 상태입니다.

전투 밸런스 확인 필요

수치, 난이도, 효율, 전투 흐름 등 밸런스 확인이 필요한 상태입니다.

보상 확정 필요

이벤트, 미션, 상품, 콘텐츠 보상 구성을 확정해야 하는 상태입니다.

자동 표시 기준

오늘 확인 필요

오늘 아직 확인하지 않은 이슈 있음, 완료 확인 대기, 2일 이상 확인 없음 티켓에 표시됩니다. [변동 없음]이나 [상태 변경]으로 확인하면 사라집니다.

2일 이상 확인 없음

완료가 아닌 티켓의 마지막 확인 시간이 2일 이상 지나면 자동으로 표시됩니다. 상태가 그대로라면 [변동 없음]으로 확인 시간만 갱신해도 됩니다.

개발 마감 강조

개발 마감 7일 전부터 상단 빌드 영역과 D-day가 강조됩니다. 마감이 지난 경우에는 더 강하게 표시됩니다.

완료 티켓

완료된 티켓은 목록에 남겨두되 시각적으로 한 단계 낮춰 보여줍니다. 필요한 경우 나중에 완료 필터로 다시 확인합니다.

일일 업무 공유와 변동 없음

퇴근 전 [일일 업무 공유]를 누르면 내 담당 티켓의 현재 상태가 공유용 문장으로 정리됩니다. 정상 티켓은 짧게 정리되고, 이슈가 있는 티켓은 상태, 다음 액션, 메모, 작업 담당/파트, 업무 링크가 함께 포함됩니다.

[변동 없음]은 상태와 메모를 바꾸지 않고 마지막 확인일만 갱신합니다. 새로 적을 내용은 없지만 담당자가 해당 티켓을 확인했다는 의미로 사용합니다.
