# Google Sheet 연결 설정 가이드

이 문서는 현재 정적 화면을 Google Sheet 저장소와 연결할 때 따라가는 순서입니다.

## 1. Google Sheet 만들기

새 Google Sheet 파일을 만들고 아래 5개 시트 탭을 만듭니다.

- `Settings`
- `Builds`
- `Members`
- `Tickets`
- `Checkins`

초기 컬럼은 `sheet-templates` 폴더의 CSV 파일을 참고해 붙여 넣습니다.

## 2. Apps Script 붙이기

Google Sheet에서 `확장 프로그램 > Apps Script`를 열고, `apps-script/Code.gs` 내용을 붙여 넣습니다.

같은 Google Sheet에 묶인 Apps Script라면 `SPREADSHEET_ID`는 비워둬도 됩니다.

## 3. 웹 앱으로 배포

Apps Script에서 배포 설정을 아래처럼 잡습니다.

- 유형: 웹 앱
- 실행 사용자: 나
- 액세스 권한: 회사 계정에서는 `회사 전체 권한`, 개인 계정에서는 `링크가 있는 사용자`

배포 후 나오는 Web App URL을 복사합니다.

## 4. 보드에서 연결

보드 상단의 `브라우저 저장` 버튼을 누릅니다.

현재 팀용 Web App URL은 기본값으로 들어가 있습니다.

1. `연결 확인`을 누릅니다.
2. 처음에는 `현재 데이터를 시트로 저장`을 눌러 현재 샘플/로컬 데이터를 시트에 올립니다.
3. 다른 PC나 팀원이 열 때는 `시트에서 불러오기`를 누릅니다.

브라우저에 저장된 데이터가 없는 첫 실행에서는 기본 URL을 이용해 시트 데이터를 자동으로 한 번 불러옵니다.

연결 후에는 메인 화면 오른쪽의 `시트 불러오기`, `시트에 저장` 버튼으로 바로 동기화할 수 있습니다.

## 연결 확인이 실패할 때

`Apps Script URL을 불러오지 못했습니다`가 뜨면 보드가 Google Sheet 내용까지 도달하지 못한 상태입니다.

우선 데이터 연결 창에서 `테스트 URL 열기`를 눌러 확인합니다.

- `연결 정상입니다` 안내 화면이 보이면 URL 접근은 가능한 상태입니다.
- Google 로그인 화면이나 권한 안내가 보이면, 같은 브라우저에서 회사 Google 계정으로 로그인한 뒤 다시 시도합니다.
- 404 또는 찾을 수 없음이 보이면 배포 URL이 잘못됐을 가능성이 높습니다. `/exec`로 끝나는 Web App URL인지 확인합니다.
- Apps Script 코드를 수정한 뒤에는 `배포 > 배포 관리 > 수정 > 새 버전`으로 다시 배포한 URL을 사용합니다.

## 참고

- 현재 사용자, 나이트 모드, 화면 필터는 각자 브라우저에 저장됩니다.
- 빌드, 팀원, 티켓, 체크인 기록은 Google Sheet에 저장되는 공용 데이터입니다.
- 샘플 데이터의 `example.com` 주소는 예시용으로 예약된 도메인입니다. 실제 업무 링크나 내부 정보가 포함된 주소가 아닙니다.
- 시트에 저장할 때는 먼저 시트 최신 데이터를 읽고, 수정 시간이 더 최신인 항목을 기준으로 합쳐 저장합니다. 같은 티켓을 동시에 고치면 더 최근 수정이 남습니다.
- 연결에 실패해도 기존 브라우저 저장 데이터는 유지됩니다.
